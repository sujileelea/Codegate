# 데이터 아키텍처 및 보안 설계안

> 확정 기술 스택 기준으로 전면 개정한 판본이다. 이전 판의 AWS Cognito · API Gateway · Lambda · Bedrock 구성은 폐기되었다(인증 Firebase, 서버 FastAPI, AI Gemini/Qwen으로 확정). API 계약의 정본은 [API_SPEC.md](./API_SPEC.md), 보안 통제 목표의 정본은 [SECURITY.md](./SECURITY.md)다.

## 1. 설계 목표

똑똑용돈은 시니어의 미션·게임 참여를 가족 돌봄 데이터로 연결한다. 앱은 가볍고 오프라인에서도 안정적으로 동작해야 하며, 가족 공유와 AI 분석에 필요한 데이터만 클라우드에 저장한다.

- **로컬 우선 경험**: 게임 진행, 미전송 결과 큐는 기기에서 즉시 처리한다.
- **클라우드 기준 데이터**: 가족 관계, 미션 완료, 활동 요약, 보상 상태, AI 리포트는 서버가 최종 기준이다.
- **최소 수집**: 원본 행동 로그·위치 경로·의료 기록은 저장하지 않는다. 회원가입 수집 항목은 실명·이메일·전화번호·성별·생년월일(+역할·타임존)로 확정.
- **비진단 원칙**: AI 결과는 가족 돌봄을 위한 활동 변화 참고 정보이며 의료적 진단이나 질병 판정이 아니다.

## 2. 전체 구조 (확정 스택)

```text
[SwiftUI iOS 앱]
  ├─ 로컬: 진행 중 게임 세션 / 오프라인 동기화 대기열(멱등키 포함)
  └─ Firebase Auth 로그인 (Apple / 카카오 oidc.kakao) → ID 토큰
          ↓ HTTPS
[FastAPI (Python) 서버]
  ├─ Firebase ID 토큰 검증 (uid만 신뢰)
  ├─ 가족 권한 검증 + 감사 로그
  ├─ 활동·보상 비즈니스 로직 (포인트는 서버 확정)
  ├─ 통계 로직 (변화 패턴·케어 상태 — AI 미사용)
  ├─ AI Gateway (배치: Gemini 3.5 Flash / Qwen 3 32B — OpenAI 호환 API)
  └─ PostgreSQL (5스키마: identity/activity/reward/ai/audit)
          ↓
[시니어 홈·달력 / 자녀 대시보드 / 맞춤 추천 / 주간 AI 리포트]
```

| 영역 | 구성 | 역할 |
| --- | --- | --- |
| 앱 | Swift + SwiftUI | 시니어 미션·게임, 자녀 대시보드 |
| 인증 | Firebase Authentication (Apple + 카카오 OIDC) | 로그인·JWT 발급. 카카오 전용 백엔드 엔드포인트 없음 |
| API 서버 | Python + FastAPI | 토큰 검증, 권한, 비즈니스 로직, AI Gateway |
| 데이터베이스 | PostgreSQL (pgvector는 후속 확장) | 5스키마 분리 저장 |
| 파일 스토리지 | 오브젝트 스토리지 — **미정**(배포 플랫폼과 함께 확정, API_SPEC 부록 A-7 ⑨) | 음성 응원 편지 오디오 전용 — 암호화 at rest + 24h 수명주기 + 서명 URL |
| AI | Gemini 3.5 Flash + gemini-embedding-001(768차원) + Qwen 3 32B(OpenAI 호환 — 운영 OpenRouter, 설정으로 Groq 전환 가능) + 통계 로직 | 태스크별 매핑은 §5 |
| CI/CD | GitHub Actions | 테스트·시크릿 스캔·배포 |
| 배포 플랫폼 | **AWS EC2(Docker Compose) + RDS PostgreSQL 16 — 운영 적용** | 저장 암호화(RDS·EBS), 도메인 TLS 종단(공인 인증서 자동 갱신), 인바운드 최소 개방. 접속 정보는 git 밖 로컬 파일 참조 |
| 비밀 관리 | **SSM Parameter Store(SecureString) 원본 + 서버 env 운영 사본** | 앱 코드의 KMS 직접 로드·키 로테이션 절차는 잔여(보안 문서.md §3) |

클라이언트가 요청 본문에 담은 사용자 ID나 가족 ID는 권한 판단의 근거로 신뢰하지 않는다. 서버는 Firebase ID 토큰의 `uid`만 신뢰한다.

### 저장하지 않는 데이터

- 걸음 수·센서 데이터 (**걸음 기능 제거 확정** — HealthKit 연동 없음)
- GPS 이동 경로, 분 단위 센서 원본, 의료기관 진료·처방 기록
- 불필요한 음성·카메라 원본 (예외: **음성 응원 편지 오디오**는 전달 콘텐츠로서 위 표의 조건 — 암호화·24h 파기·분석 금지 — 하에만 저장)
- 실제 금융결제 정보 (충전·구독 카드 정보는 Apple이 보관, 서버는 StoreKit 트랜잭션 ID만)
- 실물 선물 배송 주소 (demo 이행 한정 — 실배송 도입 시 별도 보안 검토 선행)
- 앱 코드·로컬 저장소·GitHub에 포함된 API 키와 비밀번호
- 이름·연락처 등 직접 식별정보가 포함된 AI 프롬프트 원문 (입력 해시만 기록)
- **똑똑이 음성 일기의 음성 원본** — STT는 기기 내 처리, 오디오 파일은 서버로 전송·저장하지 않는다(업로드 API 자체가 없음). 기기에서도 전사 확인 후 즉시 폐기
- 응원 편지·전사의 분석 산출물 — 응원 콘텐츠는 어떤 AI·통계 분석 파이프라인에도 투입하지 않는다(발화 분석은 시니어 본인 동의 하의 음성 일기 전용)

## 4. 활동 데이터와 동기화 규칙

### 게임·미션 (시드: 미션게임 기획서 최종 6종)

- 게임 중 원본 터치와 세부 시도는 로컬 세션 안에서만 사용한다.
- 완료 시 `정답률`, `문항별 반응시간`, `오류 횟수`, `재시도`, `힌트 사용`, `참여 시각`, `성공 여부` 요약만 서버로 전송한다(반응시간·힌트 계측은 전 게임 필수 — 확정).
- 성공/실패 판정과 포인트 지급은 서버가 `raw_payload` 재계산으로 최종 확정한다.
- 오프라인 완료 건은 멱등키와 함께 대기열에 저장하고, 재연결 시 중복 없이 전송한다(`replayed` 응답으로 이중 적립 차단).

### 체크인

- 기분(1~5)·수면 품질(1~3)을 하루 1회 upsert. `note_text`는 임의 개인정보 입력 가능성이 있어 암호화 저장하고 LLM 입력에서 제외한다.

### 똑똑이 음성 일기 (API_SPEC §4·§1.9 특칙)

- 녹음·STT는 전부 기기 내에서 처리한다. 서버에는 **전사 텍스트 + 기기 산출 발화 요약 수치**(발화 시간·속도·멈춤·재시작·STT 신뢰도)만 하루 1회 upsert로 전송한다.
- 전사 텍스트는 `voice_diary` 명시 동의(opt-in) 후에만 수집하고 **DEK 암호화 컬럼**에 저장한다. 서버 분석 지표(반복 문장 비율·어휘 다양성·필러 빈도·조리성 신호)는 일 배치가 산출하며, 보존 기간 만료 시 원문은 파기하고 파생 지표만 남긴다.
- **노출 경계**: 전사 원문은 기본적으로 시니어 본인 API에만 응답한다. **시니어가 공유 설정(`voice_diary_share_with_family`)을 켠 경우에만** 같은 가족 guardian이 원문을 열람할 수 있고(열람마다 감사 로그), 끄는 즉시 회수된다. 공유 여부와 무관하게 guardian에게는 참여 여부·정서 신호·발화 특징 **변화 요약**이 리포트·패턴 화면에 비진단 문구로 제공된다.
- 시니어 본인 삭제·동의 철회 시 전사와 파생 지표를 함께 삭제한다.

### 음성 응원 편지 (API_SPEC §3·§1.9 특칙)

- 자녀가 녹음한 오디오(AAC ≤60초·5MB)와 기기 내 STT 전사를 multipart로 업로드한다. 오디오는 오브젝트 스토리지에 암호화 저장, 전사는 DB 암호화 컬럼에 저장한다.
- **전달 콘텐츠 원칙**: 응원 오디오·전사는 재생·표시 목적 외 어디에도 쓰지 않는다 — 발화 특징·정서 분석 파이프라인 투입 금지, LLM 입력 제외. 유일한 예외는 발송 시점 1회의 **안전 필터**: 서버 내 금칙어·패턴 사전으로 전사만 검사(욕설·모욕 / 위협 / 위험 행위 조장 / 금전 요구 의심), 감지 시 저장 없이 거부, 로그는 통과/차단 여부만.
- 보존: **24시간 휘발 또는 영구 보관 둘 중 하나** — 발송 시점의 발신자 티어로 확정되어 스키마에 저장되며 이후 변하지 않는다(무료 발신분은 수명주기 규칙 + 파기 배치로 자동 파기·사후 전환 불가, 프리미엄 발신분은 구독 해지 후에도 유지).
- 접근: 수신 시니어(편지함)와 발신자(자기 발신분)만. 오디오는 15분 서명 URL로만 제공하고 URL을 로그에 남기지 않는다.
- 익명 발송은 표시상 익명이며 서버는 발신자를 기록한다(남용 대응·법적 요청 대비 — 개인정보처리방침 명시).

## 5. AI 데이터 처리 원칙 (확정 조합)

```text
활동·게임 결과 요약
  ↓
통계 로직(코드): 변화율, 연속 미참여, 평균 반응시간 변화, 케어 상태·이상 신호
  ↓
AI: 활동 데이터 구조화(Gemini) / 추천·가족 리포트 문장 생성(Qwen)
  ↓
시니어 개인화 / 자녀 주간 요약
```

| 태스크 | 주기 | 처리 |
| --- | --- | --- |
| 활동 데이터 구조화 (attempt 한 줄 요약) | 결과 제출 시마다 | **Gemini 3.5 Flash** (Google AI Studio — 2.5는 신규 발급 키에서 생성 불가 실측) |
| 인사이트·미션 임베딩 | 추천 배치 내 | **gemini-embedding-001** 768차원 축소+정규화 (text-embedding-004는 신규 키 미제공) |
| 변화 패턴 분석 (케어 상태·패턴 통계·이상 신호) | 일 배치 | **통계 로직(코드)** — AI 미사용 |
| 케어 알림 생성·푸시 발송 (무활동 연속 2일 `caution`·3일 `alert`+APNs 푸시·3일 간격 재알림·재개 `info` — API_SPEC §8 룰 정본) | 일 배치 | **통계 로직(코드)** — 티어 무관(안전 기능), 푸시 페이로드는 관계 호칭 수준만(실명·활동 상세 금지) |
| 맞춤 활동 추천 | 하루 1회 배치 | **Qwen 3 32B** (OpenAI 호환 API — 운영 OpenRouter, 설정으로 Groq 전환 가능) |
| 기피 게임 보너스 책정 | 하루 1회 (추천 배치에 포함) | **통계 로직**(기피 지표) + **Qwen 3 32B**(보너스 결정 — 기본 보상 50%·2게임 상한) |
| 가족 주간 리포트 | 주 1회 배치 + 수동 | **Qwen 3 32B** (OpenAI 호환 API) — 비진단 톤 프롬프트 강제 + 사람 검수 |
| 음성 일기 발화 특징 산출 | 일 배치 | **통계 로직(코드)** — 전사에서 수치만 산출, 외부 전송 없음 |
| 음성 일기 정서·주제 구조화 | 일 배치 | **Gemini 3.5 Flash** — **PII 스크러빙 후** 가명 ID로만 전송, 원문 프롬프트 미저장(해시만) |

- 앱은 AI 모델을 직접 호출하지 않는다. FastAPI의 AI Gateway만 외부 모델을 호출한다(요청 경로 동기 호출 금지 — 배치 + 폴백).
- AI 입력은 익명 내부 ID(UUID)와 기간별 활동 요약으로 제한한다. 실명·이메일·전화번호·생년월일·가족관계는 전달하지 않는다. 리포트 문단은 `{name}` 플레이스홀더로 생성 후 서버가 치환한다.
- OpenAI는 스택에 포함되나 용도 미정(임베딩·pgvector 추천 고도화 후보 — API_SPEC 부록 A-7). 무료 티어(Google AI Studio·OpenRouter/Groq)의 데이터 활용 약관 검토는 정식 서비스 전 필수.

AI 리포트 표현 원칙:

> 활동 참여와 반응 패턴의 변화를 가족이 이해하기 쉬운 형태로 요약하며, 의료적 진단이나 질병 판정이 아닌 가족 돌봄을 위한 참고 지표로 활용한다.

## 7. MVP 구현 우선순위

1. Firebase 로그인(Apple+카카오) → `POST /auth/session` 연결, 가입 정보(실명·이메일·전화번호·성별·생년월일)·약관·역할 수집.
2. PostgreSQL 5스키마에 사용자·가족 연결·미션·활동 요약·포인트 원장 저장.
3. SwiftUI에서 게임 완료 → 결과 제출(멱등) → 서버 판정·적립 플로우.
4. 자녀 대시보드 1콜 + 주간 지표·활동 달력.
5. AI Gateway: 활동 구조화(Gemini)와 주간 리포트(Qwen) 중 리포트 우선 1종 검증.
6. 게임 신규 4종(숫자·기호, 다른 그림, 신호 탭 + 목업 2종 완성)은 `sequence_recall` 파이프라인 검증 후 순차 추가.

## 8. MVP 범위 판단

이번 MVP는 아래 플라이휠이 실제로 이어지는지 검증하는 데 집중한다.

```text
미션·게임 참여 → 활동 요약 저장 → 통계·AI 요약 → 가족 대시보드 → 재참여
```

`pgvector`(OpenAI 임베딩), 파일 저장소, 복잡한 분석 파이프라인, 실제 금융 지급(충전 실결제·구독 실결제·출금 실이체·기프티콘 실발급·실물 배송)은 후속 확장 항목으로 둔다. 프리미엄 구독(API_SPEC §11)은 **데모 구독 상태로 게이트 플로우만 검증**한다. 똑똑이 음성 일기는 보안 게이트(보안 문서.md §5.1 V-1~V-8) 충족을 전제로 구현하되 **LLM 정서 구조화는 미학습 보장 약관 확인 전까지 비활성**(통계 지표만 운영)이며, 분담결제 정산·B2B/B2G 집계 제공은 각각 실결제·계약 확정 후 진행한다.
