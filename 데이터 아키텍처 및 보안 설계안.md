# 데이터 아키텍처 및 보안 설계안

## 1. 설계 목표

똑똑용돈은 시니어의 미션·게임 참여를 가족 돌봄 데이터로 연결한다. 앱은 가볍고 오프라인에서도 안정적으로 동작해야 하며, 가족 공유와 AI 분석에 필요한 데이터만 클라우드에 저장한다.

- **로컬 우선 경험**: 게임 진행, 당일 활동 집계, 미전송 데이터는 기기에서 즉시 처리한다.
- **클라우드 기준 데이터**: 가족 관계, 미션 완료, 활동 요약, 보상 상태, AI 리포트는 서버가 최종 기준이 된다.
- **최소 수집**: 원본 행동 로그·위치 경로·의료 기록은 기본적으로 저장하지 않는다.
- **비진단 원칙**: AI 결과는 가족 돌봄을 위한 활동 변화 참고 정보이며 의료적 진단이나 질병 판정이 아니다.

## 2. AWS 기반 전체 구조

```text
[SwiftUI iOS 앱]
  ├─ 로컬: 진행 중 게임 / 당일 활동 집계 / 동기화 대기열
  └─ Cognito 로그인 토큰
          ↓
[API Gateway]
          ↓
[FastAPI on AWS Lambda]
  ├─ 인증·가족 권한 검증
  ├─ 활동·보상 비즈니스 로직
  ├─ AI Gateway
  └─ RDS PostgreSQL
          ↓
[자녀 대시보드 / 맞춤 미션 / 주간 AI 리포트]
```

| 영역 | 구성 | 역할 |
| --- | --- | --- |
| 앱 | Swift + SwiftUI | 시니어 미션·게임, 자녀 대시보드 |
| 인증 | Amazon Cognito | 부모·자녀 로그인 및 JWT 발급 |
| API | API Gateway + FastAPI | 인증 확인, 요청 제한, 비즈니스 로직 |
| 서버 | AWS Lambda | MVP용 서버리스 FastAPI 실행 |
| 데이터베이스 | RDS PostgreSQL | 관계형 기준 데이터 저장 |
| 비밀 관리 | AWS Secrets Manager | DB 비밀번호와 외부 AI API 키 보관 |
| AI | Bedrock Claude + Gemini API | 활동 요약, 추천, 가족 리포트 |
| 모니터링 | CloudWatch | 오류·권한 실패·AI 호출 이벤트 확인 |

인증은 Cognito로 단일화한다. iOS 앱은 Cognito 토큰을 전송하고, API Gateway와 FastAPI가 토큰·권한을 검증한다. 클라이언트가 요청 본문에 담은 사용자 ID나 가족 ID는 권한 판단의 근거로 신뢰하지 않는다.

## 3. 로컬·클라우드 데이터 구분

| 데이터 | 저장 위치 | 목적 | 보존 원칙 |
| --- | --- | --- | --- |
| 테마·화면 상태 | iOS 로컬 | 빠른 UX | 필요 시 갱신 |
| 진행 중 게임 터치·시도 | iOS 로컬 | 즉시 반응·오프라인 진행 | 세션 종료 후 삭제 |
| 당일 걸음 수 합계 | iOS 로컬 → 클라우드 | 일일 활동 집계 | 전송 확인 후 로컬 원본 정리 |
| 미전송 활동 요약 | iOS 동기화 대기열 | 네트워크 복구 후 재전송 | 전송 성공 후 삭제 |
| 미션 완료·게임 결과 요약 | RDS PostgreSQL | 가족 대시보드·변화 분석 | 제품 보존 정책에 따름 |
| 가족 연결·동의·열람 권한 | RDS PostgreSQL | 접근 제어 | 해제·탈퇴 시 즉시 반영 |
| 포인트·보상 상태 | RDS PostgreSQL | 서버 기준 확정 | 감사 가능하게 보존 |
| AI 추천·주간 리포트 | RDS PostgreSQL | 개인화·가족 열람 | 제품 보존 정책에 따름 |
| API 키·DB 비밀번호 | Secrets Manager | 서버 운영 | 앱·저장소·로그 노출 금지 |

### 저장하지 않는 데이터

- GPS 이동 경로, 분 단위 센서 원본, 의료기관 진료·처방 기록
- 불필요한 음성·카메라 원본
- 앱 코드·로컬 저장소·GitHub에 포함된 API 키와 비밀번호
- 이름·연락처 등 직접 식별정보가 포함된 AI 프롬프트 원문

## 4. 활동 데이터와 동기화 규칙

### 걸음 수

걸음 수는 원본 센서 로그 대신 하루 단위 합계만 다룬다.

```text
user_id / local_date / steps / source / synced_at
```

1. 앱이 당일 걸음 수를 로컬에서 집계한다.
2. 날짜가 바뀌면 전날 값을 하루 요약으로 마감한다.
3. 온라인이면 즉시 서버에 전송하고, 실패하면 동기화 대기열에 보관한다.
4. 서버는 `사용자 ID + 날짜 + 출처`로 중복을 제거한다.
5. 서버 수신이 확인되면 로컬 원본은 삭제하고 최근 요약 캐시만 남긴다.

걸음 수는 초기 MVP에서 실제 용돈 지급의 단독 근거로 사용하지 않는다. 기기 차이, 수동 입력, 동기화 오류 가능성을 고려해 참여 변화 참고 지표로 우선 활용한다.

### 게임·미션

- 게임 중 원본 터치와 세부 시도는 로컬 세션 안에서만 사용한다.
- 완료 시 `정답률`, `반응시간`, `오류 횟수`, `재시도 여부`, `참여 시각`, `완료 여부` 같은 요약만 서버로 전송한다.
- 미션 완료와 포인트 지급 상태는 서버가 최종 확정한다.
- 오프라인 완료 건은 대기열에 저장하고, 재연결 시 중복 없이 전송한다.

## 5. AI 데이터 처리 원칙

```text
활동·게임 결과 요약
  ↓
통계 로직: 변화율, 연속 미참여, 평균 반응시간 변화 계산
  ↓
AI: 맞춤 미션 문장·가족 리포트 생성
  ↓
시니어 개인화 / 자녀 주간 요약
```

- **통계 로직**은 코드로 계산한다. 수치 비교와 추세 계산은 AI가 아닌 서버 로직이 담당한다.
- **Gemini Flash**는 자주 발생하는 활동 데이터 구조화 작업에 사용한다.
- **Claude Haiku**는 일일 맞춤 미션·피드백 생성에 사용한다.
- **Claude Sonnet**은 가족이 읽는 주간 리포트처럼 품질이 중요한 문서 생성에 사용한다.
- 앱은 AI 모델을 직접 호출하지 않는다. FastAPI의 AI Gateway만 외부 모델을 호출한다.
- AI 입력은 익명 내부 ID와 기간별 활동 요약으로 제한한다. 이름, 전화번호, 정확한 위치, 가족관계는 전달하지 않는다.

AI 리포트 표현은 다음 원칙을 지킨다.

> 활동 참여와 반응 패턴의 변화를 가족이 이해하기 쉬운 형태로 요약하며, 의료적 진단이나 질병 판정이 아닌 가족 돌봄을 위한 참고 지표로 활용한다.

## 6. 보안 크로스체크

| 확인 항목 | 적용 원칙 | MVP 점검 기준 |
| --- | --- | --- |
| 인증 | Cognito 단일 인증 | 토큰 없는 API 요청 거부 |
| 권한 | 가족 연결·동의 상태를 서버에서 검증 | 연결되지 않은 가족은 데이터 조회 불가 |
| 로컬 저장 | 민감정보는 Keychain 사용, 일반 캐시는 최소화 | UserDefaults에 토큰·건강 데이터 미저장 |
| 전송 | HTTPS/TLS 사용 | 평문 API·외부 HTTP 호출 금지 |
| DB | RDS 암호화 및 비공개 네트워크 구성 | DB 포트를 인터넷에 직접 노출하지 않음 |
| 비밀값 | Secrets Manager 보관 | 앱·GitHub·로그에 키가 없는지 확인 |
| 개인정보 | 프로필과 활동 데이터를 논리 분리 | 활동 테이블에 이름·연락처 미저장 |
| AI | 서버 호출·최소 데이터 전달 | 직접 식별정보와 원본 로그 미전송 |
| 로그 | 오류·권한 실패 기록 | 로그에 토큰·전화번호·활동 원문 미기록 |
| 무결성 | 서버가 포인트·보상을 확정 | 클라이언트 포인트 값을 그대로 저장하지 않음 |
| 삭제 | 로그아웃·연결 해제 시 로컬 캐시 삭제 | 가족 해제 직후 열람 권한 회수 |

## 7. MVP 구현 우선순위

1. SwiftUI에서 미션·게임 완료 후 활동 요약을 생성한다.
2. Cognito 로그인과 FastAPI 토큰 검증을 연결한다.
3. PostgreSQL에 사용자, 가족 연결, 미션 완료, 활동 요약, 포인트 상태를 저장한다.
4. 자녀 대시보드에서 연결된 시니어의 주간 참여 요약을 보여준다.
5. FastAPI AI Gateway가 활동 요약을 바탕으로 맞춤 미션 또는 주간 리포트 한 가지를 생성한다.
6. 실제 걸음 수 연동은 사용자 동의·권한 화면과 보존 정책을 마련한 뒤 다음 단계에서 적용한다.

## 8. MVP 범위 판단

이번 MVP는 모든 AWS 서비스를 완벽히 구현하는 것이 아니라, 아래 플라이휠이 실제로 이어지는지 검증하는 데 집중한다.

```text
미션·게임 참여 → 활동 요약 저장 → AI 피드백 → 가족 대시보드 → 재참여
```

`pgvector`, 파일 저장소, 복잡한 분석 파이프라인, 실제 금융 지급은 후속 확장 항목으로 둔다.
