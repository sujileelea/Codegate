# 기능 명세서

> 정본 관계: 화면·API 계약은 [API_SPEC.md](./API_SPEC.md) v2.1(부록 B 매핑), UI 상세는 [UI 구성 가이드](./UI%20구성%20가이드.md), 보안 통제는 [보안 문서.md](./보안%20문서.md), 스토리·인수 조건은 [유저스토리.md](./유저스토리.md)(US-xx). 이 문서는 기능 단위의 목적·범위 정의를 담당한다.

**기능 1. 시니어 참여 홈** (S-1 · US-03)

- 목적: 시니어가 매일 앱에 쉽게 진입해 오늘 할 일을 고민 없이 확인하고 바로 시작하도록 유도
- 주요 기능: 똑똑이 인사말(탭 → 음성 일기 진입), 오늘의 미션 카드(AI 추천 1위 — 인지 라벨 칩·보너스 배지), 오늘 참여 링(x/5), 체크인 배너(+10P), 우체통(응원 편지 도착 시 흔들림), 다른 활동 리스트, 잔액·주간 목표
- 핵심 가치: 용돈을 트리거로 "오늘 무엇을 하면 되는지"를 직관적으로 안내하고, 똑똑이·우체통이라는 정서 접점으로 매일 앱을 열 이유를 더함
- 데이터 연결: 홈 1콜 — 날짜("오늘" 판정 포함)·포인트·진행률은 전부 서버 값

**기능 2. 미션형 인지 게임** (S-3~S-7 · US-04)

- 목적: 병원식 검사보다 부담이 적은 게임 참여를 만들고, 개인별 활동 변화를 관찰할 수 있는 반복 수행 데이터를 축적
- 확정 게임 시드 6종(미션게임 기획서 최종 — 일일 5개 배정, 보상은 인지 기여도 순위 비례):

| 게임 | 인지 라벨 | 기본 보상 | 관찰 데이터 |
| --- | --- | --- | --- |
| 숫자·기호 짝맞추기 (`digit_symbol`) | 순발력 | 50P | 처리 속도·시각 주의 |
| 다른 그림 찾기 (`spot_difference`) | 관찰력 | 45P | 시지각·주의 |
| 순서 기억하기 (`sequence_recall`) | 기억력 | 40P | 작업 기억 |
| 신호에 맞춰 누르기 (`signal_tap`) | 집중력 | 35P | 반응 억제·지속 주의 |
| 단어·그림 분류하기 (`word_sort`) | 언어력 | 30P | 언어·범주화 |
| 일상 순서 맞추기 (`daily_order`) | 판단력 | 25P | 실행 기능 |

- 계측(전 게임 필수): 정답률, 문항별 반응시간, 오류 횟수, 재시도, 힌트 사용, 참여 시각 — 성공/실패 판정과 포인트는 서버가 확정
- 참여 유도: AI가 기피 게임에 보너스 포인트를 책정(기본 보상 50% 상한·동시 2게임)해 활동 편식을 완화
- 핵심 가치: 게임 데이터는 의료적 진단이 아닌 개인별 참여 패턴·활동 변화 관찰 지표로만 활용(비진단 원칙)

**기능 3. 용돈 기반 보상 구조** (S-8/S-12/S-17·F-4/F-10 · US-07·US-08)

- 목적: 미션 참여를 지속시키는 동기를 제공하고, 자녀의 송금을 부모님의 행동 트리거로 전환
- 보상 흐름: 자녀가 가족 지갑에 충전(인앱 결제) → 시니어가 미션으로 적립(1P=1원 고정) → 출금·상점·가족 선물로 사용
- 규칙: 자녀가 주간 목표(100~2,000P)·목표 보너스·연속 참여 보너스·주간 상한을 설정하며, 시니어의 주간 목표 화면과 같은 데이터 원천을 공유
- 핵심 가치: 용돈을 단순 송금이 아니라 지속 참여를 유도하는 보상 설계로 활용
- MVP 범위: 충전·출금·기프티콘은 데모 상태 — 실결제 전환은 자금 흐름 남용 방어 게이트(보안 문서.md §4.7 M-1~M-8) 충족 후

**기능 4. 가족 대시보드·케어 알림** (F-1/F-2/F-5/F-7/F-9/F-12·S-19 · US-10·US-11·US-12)

- 목적: 자녀가 부모님의 참여 현황과 변화 흐름을 해석 없이 한눈에 이해하고, 확인이 필요한 순간을 놓치지 않도록 지원
- 주요 기능: 대시보드 1콜(케어 상태 3단계·주간 참여율·오늘 미션·최근 활동), 지표 추이 3종(정답률/반응시간/참여 빈도), 활동 달력, 미션 히스토리, 참여 패턴 요약, **케어 알림·푸시**(무활동 연속 2일 `caution` / 3일 `alert`+푸시 / 재개 `info` — 앱을 열지 않아도 도달, 구독 티어 무관)
- 핵심 가치: 원시 기록 나열이 아닌 가족 케어 인사이트 — 평소와 다른 변화가 보일 때 자연스러운 안부 확인으로 연결
- 데이터 표현: 진단·위험 판정이 아닌 참여 증감·선호 활동·수행 변화 중심(비진단 톤), 케어 상태·이상 신호는 통계 로직(코드)으로 산출

**기능 5. 개인화 AI 분석 및 추천** (F-3·S-1 추천 · US-12)

- 목적: 누적 활동 데이터를 가족이 이해할 수 있는 요약과 시니어별 맞춤 추천으로 전환
- 확정 구성: 활동 데이터 구조화 = Gemini 3.5 Flash / 맞춤 추천·주간 리포트 = Qwen 3 32B / 변화 패턴·케어 상태 = 통계 로직(코드, AI 미사용) / 추천 임베딩 = gemini-embedding-001(768차원)
- 개인정보 원칙: 외부 AI에는 가명 ID와 활동 요약만 전송(실명·연락처 미전송), 리포트 이름은 서버가 치환, 프롬프트 원문 미저장(해시만)
- 안전 기준: 질병·치매 진단이 아닌 활동 참여·수행 변화의 요약·추천에 한정 — 진단 어휘 필터 + 리포트 사람 검수

**기능 6. 가족 소통 — 음성 응원 편지·형제자매 연결** (F-1·S-1/S-21 · US-02·US-15·US-16)

- 목적: 용돈·데이터 중심의 가족 연결을 정서적 소통으로 확장하고, 형제자매 유입으로 가족 단위 리텐션을 강화
- 주요 기능: 자녀가 똑똑이를 통해 **목소리로** 응원 편지를 보냄(60초 녹음 + 기기 내 STT 전사, 익명/실명 선택) → 부모님 홈의 **우체통이 귀엽게 흔들리며** 도착 알림 → 열면 **편지지 UI**에 전사 편지가 펼쳐지고 음성이 재생됨, 편지함에서 계속 접근. 기존 초대 플로우를 재사용한 형제자매(guardian) 추가 연결, 가족 지갑 충전 기여 내역(충전자별) 표시
- 보존 정책: **24시간 휘발 또는 영구 보관 둘 중 하나** — 발송 시점의 발신자 멤버십으로 확정(무료 발신 = 24시간 후 자동 파기·사후 전환 없음, 프리미엄 발신 = 영구 보관·구독 해지 후에도 유지) — 구독 업셀 접점
- 안전 기준: 응원 오디오·전사는 전달 목적으로만 암호화 저장하고 프로파일링 분석에 투입하지 않음, 접근은 수신 부모님·발신자만, 익명은 표시상 익명(서버는 발신 기록 유지). **발송 시 안전 필터**: 욕설·모욕·위협·위험 표현·금전 요구 의심 문구를 자동 판별(서버 금칙어·패턴 검사 + 발송 전 앱 내 1차 검사)해 감지 시 발송 차단
- MVP 범위: 분담결제·N분의1 정산은 실결제 확정 전 범위 외 — 충전 기여 표시까지만

**기능 7. AI 정서·생활 동반자 '똑똑이' — 음성 일기** (S-20 · US-13·US-14)

- 목적: 돼지 마스코트 '똑똑이'를 시니어 경험의 화자로 세우고, "오늘 하루 어땠는지 똑똑이에게 들려주세요"라는 음성 대화로 일상·기분·활동 이야기를 꾸준히 기록
- 주요 기능: 홈에서 똑똑이를 눌러 음성으로 하루 이야기 → 기기 내 STT 전사 → 시니어 확인 후 전송(하루 1회), 지난 일기 열람·삭제(본인 전용), 홈 인사말·체크인의 똑똑이 대화형 UI
- 분석 데이터: 이야기 내용(정서·주제 요약)과 발화 특징 — 발화 속도·멈춤, 같은 말 반복, 이야기 개연성, 단어를 떠올리지 못하는 신호(필러·어휘 다양성) 등의 **변화 추이**
- 핵심 가치: 시니어에게는 손자에게 이야기하듯 부담 없는 기록 경험을, 가족에게는 참여·정서 변화의 참고 신호와 소통 포인트를 제공
- 안전 기준(확정 — API_SPEC §1.9 특칙): **음성 원본 서버 미저장**(on-device STT, 오디오 업로드 API 없음) · 전사 텍스트는 **별도 명시 동의(opt-in) + 암호화 저장** · 보존은 **무료 30일 후 원문 파기(파생 지표만 보존)·프리미엄 상속 시 영구**(확정) · **전사 원문은 기본 비공개 — 가족 공개 여부는 부모 사용자 본인이 설정으로 결정**(켜면 가족 열람 가능·감사 로그, 언제든 끄기 가능) · 분석 결과는 의료적 진단이 아닌 참여·정서 변화 참고 신호로만 표현(비진단 원칙) · 동의 철회 시 전체 삭제
- 데이터 활용: 축적된 발화 특징·정서 신호는 가족 주간 리포트·참여 패턴의 소통 포인트로 연결, 향후 캐릭터 구독·디지털 선물·굿즈 확장의 기반

**기능 8. 프리미엄 구독** (F-13 · US-17)

- 목적: payer(자녀)·user(부모)가 분리된 구조에서 가장 지불 의사가 명확한 자녀에게 반복과금 모델을 검증
- 주요 기능: 3티어(free/premium/premium_ai) 구독, 무료(오늘 참여·케어 상태·케어 푸시·최신 리포트 1건·7일 추이) / 프리미엄(30일 추이·리포트 아카이브·수동 생성·참여 패턴·부모님 활동 달력·케어알림 전체 히스토리·프리미엄 교육 콘텐츠 가족 상속·음성 편지 영구 보관), 페이월·잠금 UI
- 핵심 가치: 이미 만든 대시보드·리포트 위에 결제 게이트만 얹는 구조 — 신규 화면은 페이월 1개
- 안전 기준: 게이트는 서버가 강제(402), 시니어 본인의 자기 데이터 열람·케어 푸시는 티어와 무관하게 항상 제공(안전 기능에 과금 게이트 금지), 시니어에게 직접 결제 유도 금지
- MVP 범위: 데모 구독 상태로 게이트 플로우 검증 — 실결제·가격은 결정 대기(API_SPEC 부록 A-7 ②·④)

**기능 9. 포인트 상점 커머스 확장 · 가족 선물하기** (S-13 · US-09)

- 목적: 용돈 포인트 소비처를 넓혀 포인트 순환을 강화하고, 가족 간 선물 순환으로 리테일 마진·제휴(B2B) 수익의 기반을 마련
- 주요 기능: 상점 카테고리 확장(꾸미기·테마·기프티콘·교육·건강·제휴), 건기식·보청기 등 시니어 특화 상품 카테고리, 제휴 상품(`partner`) 지면 — 별도 광고 시스템 없이 상점 상품으로 흡수. **가족 선물하기**: 포인트로 현물 상품(기프티콘·건강 상품 등)을 구매해 가족 구성원에게 선물 — 시니어는 미션으로 모은 용돈으로 자녀·손주에게, 자녀는 가족 지갑 충전 잔액으로 부모님께. 선물 도착 알림 + 가족 선물 내역
- 핵심 가치: "부모님이 모은 용돈으로 손주에게 커피 한 잔" — 보상 포인트가 가족 정서 순환으로 이어지는 가족 커머스. 제휴 상품이 선물로 오가며 B2B 채널(시니어 용품·건기식 제휴)의 자연스러운 접점이 됨
- 안전 기준: 실결제 전환 시 자금 이동 채널 악용(세탁·보이스피싱) 방지를 위해 충전·현금성 선물에 한도, 신규 가족 연결 후 냉각 기간, 이상 거래 탐지를 적용 (정본: 보안 문서.md §4.7)
- MVP 범위: 실발급·실물 배송 없이 전 상품 데모 이행 상태(배송 주소 미수집) — 실결제 범위 확정 후 전환

**기능 10. 아바타 꾸미기 (개인화)** (G-2 · US-18)

- 목적: 마이페이지 아바타를 직접 꾸미는 가벼운 개인화로 애착·재방문 동기를 만들고, 포인트 소비처(상점 꾸미기 아이템)를 넓힘
- 주요 기능: **얼굴은 전 사용자 공통 고정, 머리 모양·옷차림 2개 슬롯만 선택**(G-2 화면 — 대형 미리보기 + 슬롯별 그리드). 기본 세트(슬롯별 6종 내외)는 무료, 추가 아이템은 상점 꾸미기(`deco`) 상품을 포인트로 구매해 잠금 해제. 아바타 아이템은 가족 선물 가능
- 표시 위치: 마이페이지(S-10)·설정(G-1)·가족 구성원 목록(F-6)·대시보드 부모 선택 칩(F-1)
- 핵심 가치: 시니어에게는 쉬운 2택 구조의 자기표현을, 서비스에는 미션 적립 포인트의 소액 순환 소비처를 제공
- 데이터 연결: 머리·옷 코드 2개만 저장(개인정보 아님), 서버가 소유 여부 검증
