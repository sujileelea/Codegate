# 유저 스토리

> 형식: 표준 유저 스토리 3단 구문("〈역할〉로서, 〈기능〉을 원한다. 그래야 〈가치〉를 얻는다") + **인수 조건**(Given/When/Then). 우선순위는 MoSCoW(Must/Should/Could). 화면 코드·API의 정본은 [API_SPEC.md](./API_SPEC.md) 부록 B, UI 상세는 [UI 구성 가이드](./UI%20구성%20가이드.md).
> 구성: E1~E7은 최종 사용자(자녀·부모님) 스토리 — 앱 구현·QA의 기준이므로 인수 조건까지 명세한다. E8은 내부 이해관계자(운영·검수·데이터·owner) 스토리 — 관리 도구가 아직 미설계라 요구만 간결하게 담는 백로그다.

## 30초 요약

떨어져 사는 자녀는 부모님께 용돈을 보내면서도 매일의 안부를 챙기지 못하고, 시니어는 건강·인지 관리가 필요한 걸 알면서도 검사식 헬스케어 앱을 지속하지 못한다.

똑똑용돈은 그 용돈을 행동 트리거로 바꾼다. 부모님이 매일 게임 미션·체크인·똑똑이 음성 일기에 참여하면 용돈이 보상으로 적립되고, 거기서 쌓인 활동·발화 데이터가 진단이 아닌 **변화 참고 신호**(대시보드·주간 리포트·3일 무활동 푸시)로 자녀에게 전달되며, 음성 편지와 가족 선물이 데이터 너머의 정서적 소통을 잇는다.

경쟁 서비스와 갈리는 지점은 접근 방식이다. 기존 시니어 헬스케어는 측정하라고 시키지만, 똑똑용돈은 하고 싶게 만든다. 영업 문장으로는 **"저희는 진단하지 않습니다 — 매일의 참여를 만듭니다"**가 가장 강하다. payer(자녀)와 user(부모)가 분리되어 지불 의사가 명확한 것이 수익 구조의 강점이다.

## 페르소나

| 페르소나 | 정의 |
|---|---|
| **자녀 (보호자 회원)** | 부모님께 정기적으로 용돈을 보내는 40~50대. 바쁜 일상과 물리적 거리 때문에 부모님의 매일의 활동·생활 상태를 꾸준히 확인하기 어렵고, 여러 기록을 직접 해석할 시간이 없다. 송금이 "돌봄"으로 이어지기를 원한다 |
| **부모님 (시니어 회원)** | 건강·인지 관리의 필요성은 알지만 치매 검사·병원식 헬스케어 앱은 부담스럽고 재미없어 지속하지 못하는 시니어. 재미와 보상이 있어야 꾸준히 참여하며, 검사받는 기분 없이 가족과 연결되고 싶다 |

## E1. 온보딩·가족 연결

### US-01 가입과 부모님 연결 (자녀 · Must)

**자녀로서**, Apple/카카오 로그인과 4자리 초대 코드로 부모님 계정을 연결하기를 원한다. **그래야** 복잡한 절차 없이 바로 부모님 케어를 시작할 수 있다.

- Given 신규 사용자가 When Apple 또는 카카오로 로그인하고 약관 동의(전문 열람 가능)·가입 정보(실명·이메일·전화번호·성별·생년월일)·역할을 입력하면 Then 계정이 생성되고 역할은 변경 불가로 확정된다
- Given 생년월일이 만 14세 미만이면 When 가입을 시도해도 Then 서버가 가입을 거부한다
- Given 가족을 생성한 자녀가 When 초대 코드를 발급하면 Then 4자리·24시간 유효·1회용 코드가 카카오톡 공유/QR과 함께 표시된다
- Given 부모님이 코드를 입력해 참여하면 When 자녀 화면이 초대 상태를 폴링해 Then "연결 완료"로 전환된다

관련: O-4~O-8 / `POST /auth/session`, `POST /families`, `POST /invites`, `POST /families/join`

### US-02 형제자매 초대 (자녀 · Should)

**자녀로서**, 형제자매를 같은 가족에 초대하기를 원한다. **그래야** 부모님 케어와 용돈 충전을 형제가 함께 분담할 수 있다.

- Given 기존 가족의 자녀가 When 추가 연결로 초대 코드를 공유하고 형제자매가 참여하면 Then 형제자매도 guardian 구성원이 되어 대시보드를 열람한다
- Given 형제 여럿이 충전한 가족이 When 충전 내역을 열면 Then 누가 얼마 충전했는지(`charged_by`)가 표시된다

관련: F-6, F-4/F-8 / `GET /reward-wallet/charges`

## E2. 시니어 일일 참여

### US-03 오늘 할 일 한눈에 (부모님 · Must)

**부모님으로서**, 앱을 열면 오늘의 미션·보상·내 용돈을 한 화면에서 보기를 원한다. **그래야** 무엇을 하면 되는지 고민 없이 바로 시작할 수 있다.

- Given 로그인한 시니어가 When 홈에 진입하면 Then 1회 호출로 오늘의 추천 미션(인지 라벨·보너스 배지 포함)·참여 링(x/5)·체크인 상태·우체통·잔액·주간 목표가 표시된다
- Given 자정이 지난 경우 When 홈을 새로고침하면 Then "오늘" 판정은 서버 `date` 기준으로 갱신된다(클라이언트 시계 판정 금지)

관련: S-1 / `GET /seniors/{id}/home`

### US-04 게임 미션 수행과 보상 (부모님 · Must)

**부모님으로서**, 짧은 게임 미션을 놀이처럼 수행하고 즉시 보상을 받기를 원한다. **그래야** 검사받는 부담 없이 매일 참여할 동기가 생긴다.

- Given 오늘 배정된 게임을 When 완료 제출하면 Then 성공/실패와 포인트는 서버가 판정·확정하고, 성공 시 기본 보상+기피 보너스가 분리 표시된다
- Given 실패한 미션은 When 재도전하면 Then 새 시도가 허용된다(배정 유지)
- Given 오프라인에서 완료한 경우 When 네트워크가 복구되면 Then 멱등키로 재전송되어 이중 적립 없이 반영된다
- 모든 게임은 문항별 반응시간과 힌트 사용이 자동 기록된다(전 게임 필수 계측)

관련: S-3~S-7 / `POST /attempts`, `POST /attempts/{id}/complete`

### US-05 하루 체크인 (부모님 · Must)

**부모님으로서**, 똑똑이가 묻는 말에 선택지로 답하며 기분·수면을 기록하기를 원한다. **그래야** 검사가 아니라 대화하는 기분으로 하루를 남길 수 있다.

- Given 오늘 첫 체크인이면 When 기분(1~5)·수면(1~3)을 선택 제출하면 Then +10P가 적립된다(최초 1회만)
- Given 같은 날 다시 제출하면 Then 값은 덮어써지고 추가 적립은 없다

관련: S-2 / `POST /checkins`

### US-06 활동 달력 (부모님 · Should)

**부모님으로서**, 내가 참여한 날들을 달력 스탬프로 보기를 원한다. **그래야** 쌓여가는 기록에서 성취감을 느낀다.

- Given 월간 달력에서 When 날짜 셀을 탭하면 Then 그날의 미션 5개 상태·체크인·획득 포인트 상세가 표시된다
- 달력에는 활동일·올클리어·최장 스트릭 요약이 표시된다

관련: S-19 / `GET /activity-calendar`

## E3. 보상·용돈

### US-07 충전과 보상 규칙 설정 (자녀 · Must)

**자녀로서**, 용돈을 충전하고 주간 목표·보너스 규칙을 설정하기를 원한다. **그래야** 송금이 부모님의 꾸준한 참여로 이어지도록 설계할 수 있다.

- Given 자녀가 When 금액 6종 중 선택해 인앱 결제하면 Then 서버 검증 후 가족 지갑 잔액이 갱신된다(MVP는 데모 상태 표기)
- Given 보상 규칙 화면에서 When 주간 목표(100~2,000P)·목표/연속 보너스를 저장하면 Then 부모님의 주간 목표 화면(S-17)에 동일하게 반영된다

관련: F-4/F-8/F-10 / `POST /reward-wallet/charges`, `PUT /reward-rules`

### US-08 용돈함과 출금 (부모님 · Must)

**부모님으로서**, 모은 용돈을 확인하고 출금을 신청하기를 원한다. **그래야** 참여의 결과가 실제 용돈이라는 실감을 얻는다.

- Given 용돈함에서 When 내역을 스크롤하면 Then 적립·사용 원장이 서버 값으로 표시된다(1P=1원)
- Given 출금 신청 시 When 금액을 선택하면 Then 잔액이 차감되고 신청 상태가 표시된다(MVP는 실이체 없는 데모 — 화면 고지)

관련: S-8/S-12 / `GET /wallet`, `POST /withdrawals`

### US-09 상점 구매와 가족 선물 (부모님·자녀 · Should)

**부모님으로서**, 미션으로 모은 용돈으로 자녀·손주에게 작은 선물을 보내기를 원한다. **그래야** 받기만 하는 것이 아니라 베푸는 기쁨을 느낄 수 있다.

- Given 상점에서 When 상품에 수령인(가족 구성원)을 지정해 구매하면 Then 시니어는 본인 지갑, 자녀는 가족 충전 잔액에서 차감되고 수령인에게 알림이 간다
- Given 가족이 아닌 수령인을 지정하면 Then 서버가 거부한다(403)
- 실물·기프티콘 이행은 데모 상태로 표기된다(실발급은 실결제 전환 후)

관련: S-13 / `POST /shop/items/{id}/purchase`, `GET /gifts`

## E4. 가족 케어 인사이트

### US-10 대시보드 한눈에 보기 (자녀 · Must)

**자녀로서**, 부모님의 오늘 참여·케어 상태·최근 활동을 한 화면에서 보기를 원한다. **그래야** 여러 기록을 해석하지 않고도 부모님의 최근 상태를 빠르게 이해한다.

- Given 자녀가 When 대시보드에 진입하면 Then 1회 호출로 케어 상태(3단계)·주간 참여율·오늘 미션(x/5)·최근 활동 시각·활동 리스트가 표시된다
- Given 부모님이 여러 분이면 When 상단 칩을 전환하면 Then 선택한 부모님 기준으로 갱신된다
- 케어·리포트 데이터는 기기에 저장하지 않는다(메모리 캐시만)

관련: F-1 / `GET /families/{id}/dashboard`

### US-11 무활동·이상 신호 푸시 (자녀 · Must)

**자녀로서**, 부모님 활동이 3일 이상 없거나 평소와 다른 변화가 감지되면 푸시로 알림받기를 원한다. **그래야** 앱을 열지 않아도 안부 확인이 필요한 순간을 놓치지 않는다.

- Given 부모님이 2일 연속 무활동(미션·체크인·음성 일기 모두 없음)이면 Then `caution` 케어 알림이 생성된다
- Given 3일 연속 무활동이면 When "이상 신호" 토글이 켜져 있으면 Then `alert` 알림과 함께 푸시가 발송되고, 지속 시 3일 간격으로만 재알림된다
- Given 부모님이 활동을 재개하면 Then 카운터가 리셋되고 `info` 알림이 생성된다
- 푸시 본문은 관계 호칭 수준만 담는다("어머님의 활동이 3일째 없어요" — 실명·상세 금지). 이 기능은 구독 티어와 무관하다

관련: F-5/F-11 / `GET /care-alerts`, `POST /me/devices`

### US-12 변화 추이와 주간 리포트 (자녀 · Should)

**자녀로서**, 정답률·반응시간·참여 빈도의 추이와 AI 주간 요약을 보기를 원한다. **그래야** 부모님의 변화 흐름을 근거 있게 파악하고 대화 소재를 얻는다.

- Given 추이 화면에서 When 지표 탭을 전환하면 Then 지표별 시계열이 호출된다(7일 무료, 30일은 프리미엄)
- Given 주간 리포트를 열면 Then 문단형 요약·관리 포인트·추천 활동이 비진단 톤으로 표시된다(진단·질병 표현 없음)
- Given 수동 생성을 요청하면 Then 수십 초 로딩 UI와 함께 생성되며 한도 초과 시 안내된다(429)

관련: F-2/F-3/F-9 / `GET /metrics/daily`, `GET /reports`

## E5. 정서적 소통

### US-13 똑똑이에게 하루 들려주기 (부모님 · Should)

**부모님으로서**, 똑똑이를 눌러 오늘 하루를 목소리로 들려주면 일기처럼 기록되기를 원한다. **그래야** 글자 입력 부담 없이 손자에게 이야기하듯 하루를 남길 수 있다.

- Given 첫 진입 시 When 별도 동의 화면을 확인하면 Then "목소리는 저장되지 않아요·이야기 내용만 기록돼요·가족에게 보여줄지는 내가 결정해요"가 큰 글씨로 안내된다(동의 없이는 기능 비활성)
- Given 녹음을 마치면 When 기기 내 STT 전사를 미리 확인하고 "들려주기"를 누르면 Then 전사만 전송된다(음성 파일은 서버로 가지 않음)
- Given 지난 일기 목록에서 When 삭제하면 Then 해당 일기와 파생 지표가 함께 삭제된다

관련: S-20 / `POST /voice-diaries`

### US-14 이야기 공개는 내가 결정 (부모님 · Should)

**부모님으로서**, 내 이야기 원문을 가족에게 보여줄지 직접 결정하기를 원한다. **그래야** 사생활을 지키면서 원할 때만 마음을 열 수 있다.

- Given 기본 상태에서 When 자녀가 일기 원문에 접근하면 Then 서버가 거부한다(403) — 자녀에게는 참여 여부·정서 신호·변화 요약만 제공된다
- Given 시니어가 "가족에게 보여주기"를 켜면 Then 자녀가 원문을 열람할 수 있고 열람마다 기록이 남으며, 끄는 즉시 접근이 회수된다
- 이 설정은 시니어 본인만 변경할 수 있다(자녀의 변경 시도는 403)

관련: S-20/F-1 "부모님 이야기" / `PATCH /me`(voice_diary_share_with_family)

### US-15 음성 응원 편지 보내기 (자녀 · Should)

**자녀로서**, 부모님께 내 목소리로 응원 편지를 보내기를 원한다. **그래야** 송금 알림 한 줄보다 따뜻한 마음이 전해진다.

- Given 녹음 시트에서 When 60초 이내로 녹음하고 전사를 확인 후 익명/실명을 선택해 보내면 Then 편지가 발송된다(하루 3회 한도)
- Given 전사에 부적절 표현(욕설·위협·위험·금전 요구 의심)이 있으면 Then 앱이 발송 전에 안내하고, 서버도 거부하며 저장하지 않는다
- 발송 시점에 보존이 확정된다: 무료 발신은 24시간 후 소멸(사후 전환 없음), 프리미엄 발신은 영구 보관(구독 해지 후에도 유지) — 발송 전 이를 고지한다

관련: F-1 / `POST /families/{id}/cheers`

### US-16 우체통에서 편지 받기 (부모님 · Should)

**부모님으로서**, 우체통이 흔들리면 편지지를 펼쳐 자녀 목소리를 듣기를 원한다. **그래야** 자녀의 응원을 편지 받는 설렘으로 경험한다.

- Given 새 편지가 있으면 When 홈에 진입하면 Then 우체통이 흔들리며 배지가 표시된다
- Given 우체통을 탭하면 Then 편지지가 펼쳐지는 연출과 함께 전사 편지(큰 글씨)가 표시되고 음성 재생 버튼으로 자녀 목소리를 들을 수 있다
- Given 무료 발신 편지면 Then 24시간 후 사라짐이 안내되고("편지가 하루 뒤 하늘로 돌아가요"), 영구 보관 편지는 보관 배지가 표시된다

관련: S-1/S-21 / `GET /cheer-letters`, `POST …:listened`

## E6. 수익화

### US-17 프리미엄 구독 (자녀 · Should)

**자녀로서**, 월 구독으로 심화 인사이트(30일 추이·리포트 아카이브·참여 패턴·활동 달력·편지 영구 보관)를 열기를 원한다. **그래야** 부모님의 장기 변화를 놓치지 않고 챙길 수 있다.

- Given 무료 사용자가 When 잠금 기능에 접근하면 Then 서버가 402로 거부하고 잠금 카드 → 페이월로 이동한다(잠금 판정은 서버 정본)
- Given 페이월에서 When 구독을 결제·복원하면 Then 서버 검증 후 즉시 잠금이 해제된다(MVP는 데모 구독으로 플로우 검증)
- 부모님 본인의 자기 데이터 열람과 케어 푸시는 구독과 무관하게 항상 제공된다 — 시니어 화면에는 결제 유도를 노출하지 않는다

관련: F-13, F-2/F-3/F-5/F-9/F-12 / `GET·POST /me/subscription`

## E7. 개인화·설정

### US-18 아바타 꾸미기 (부모님·자녀 · Could)

**회원으로서**, 마이페이지 아바타의 머리 모양과 옷차림을 직접 고르기를 원한다. **그래야** 내 프로필에 애착이 생기고 모은 포인트를 쓸 곳이 생긴다.

- Given 꾸미기 화면에서 When 머리/옷 슬롯의 아이템을 선택·저장하면 Then 즉시 미리보기에 반영되고 가족 화면(구성원 목록·부모 선택 칩)에도 표시된다(얼굴은 고정)
- Given 미소유 유료 아이템을 선택하면 Then 잠금·가격이 표시되고 탭 시 상점의 해당 상품으로 이동한다 — 비소유 상태로 저장 시도 시 서버가 거부한다(400)

관련: G-2/S-10/S-13 / `GET /me/avatar/catalog`, `PATCH /me`(avatar)

### US-19 알림 설정 (공통 · Must)

**회원으로서**, 알림 종류별로 받을지 말지를 직접 정하기를 원한다. **그래야** 필요한 알림만 받고 피로하지 않다.

- Given 설정에서 When 5종 토글(미션/보상/주간 리포트/이상 신호/마케팅)을 변경하면 Then 서버에 저장되고 푸시 발송 필터로 즉시 적용된다
- 마케팅 푸시는 마케팅 수신 동의까지 함께 확인된 경우에만 발송된다

관련: F-11 / `GET·PUT /me/notification-settings`

### US-20 계정과 프라이버시 통제 (공통 · Must)

**회원으로서**, 내 정보의 열람·수정·삭제와 동의 철회를 앱에서 직접 하기를 원한다. **그래야** 내 데이터에 대한 통제권을 유지한다.

- Given 가입 동의 화면에서 When 각 약관 항목을 탭하면 Then 이용약관·개인정보처리방침 전문을 큰 글씨로 열람할 수 있다
- Given 설정에서 When 프로필을 수정하거나 선택 동의(마케팅·음성 일기)를 철회하면 Then 즉시 반영되고, 음성 일기 동의 철회 시 전사·지표가 전부 삭제됨이 사전 고지된다
- Given 탈퇴 시 Then 복구 불가 안내 후 개인정보는 암호화 키 폐기로 복호화 불능 처리되고 가족 열람 권한이 즉시 소멸한다

관련: O-5b/G-1 / `PATCH /me`, `DELETE /me`

## 모든 스토리에 함께 걸리는 불변식

개별 인수 조건과 별개로, 아래는 전 스토리에 공통 적용된다. 위반하면 어떤 스토리도 "완료"가 아니다.

- **서버 값이 정본이다.** 포인트·잔액·성공 판정·"오늘"·프리미엄 잠금·편지 보존 상태·아바타 소유를 클라이언트가 자체 판정하지 않는다.
- **진단하지 않는다.** 사용자에게 닿는 모든 문구는 활동 관찰 표현만 쓴다 — "기억력이 떨어졌습니다" 류 금지, 필터와 사람 검수로 이중 방어.
- **안전 기능에 과금 게이트를 걸지 않는다.** 케어 알림·푸시, 시니어 본인의 자기 데이터 열람은 구독 티어와 무관하다. 시니어 화면에는 결제 유도를 노출하지 않는다.
- **음성의 경계.** 시니어 일기 오디오는 서버에 존재하지 않고(기기 내 STT), 자녀 편지 오디오는 전달 목적 외 어떤 분석에도 들어가지 않는다(안전 필터 1회 예외).
- **시간 축은 시니어 타임존(Asia/Seoul)이다.** "오늘"·무활동일·달력·과금 월의 해석 기준이며, 저장은 UTC다.
- **세 가지 포인트 수를 섞지 않는다.** 가족 지갑 충전 잔액 > 시니어 적립 잔액 > 현금성 사용액은 다른 수다 — "포인트가 얼마예요?"에는 어느 수인지 되물어야 한다.

## 아직 없는 것

- **실결제·실출금·실발급** — 전부 데모 상태. 전환은 자금 흐름 남용 방어 게이트(보안 문서.md §4.7) 충족이 선결 조건이다.
- **관리 도구(어드민)** — E8의 모든 스토리가 미착수. 현재 운영 문의는 DB·로그를 사람이 직접 봐야 한다.
- **음성 일기 LLM 정서 요약** — 미학습 보장 약관 확인 전 비활성. 그때까지 통계 지표만 산출되므로 리포트의 정서 서술이 빈약한 것이 알려진 한계다.
- **premium_ai 티어 실기능** — 슬롯만 예약. 페이월에서 판매 금지("준비 중" 표기).
- **on-device STT 미지원 기기 처리** — 미결정(A-7 ⑦⒜). 해당 기기에서 음성 기능이 막히는 것이 현재 동작이다.
- **분담결제·N분의1 정산** — 형제자매 유입은 되지만 정산은 없다. 실결제 확정 전까지 충전 기여 표시가 전부다.
