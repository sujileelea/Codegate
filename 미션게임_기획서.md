# 똑똑용돈 게임 기획 기록

## 최종 MVP 후보 6종 요약

최종 MVP 후보 6종은 인지훈련 연구에서 사용된 과제 구조와의 실질적 연관성을 우선 기준으로 선정한다. 구현 난이도와 데이터 구조화 가능성은 개발 순서를 정하는 보조 기준으로 활용한다.

1. 숫자·기호 짝맞추기 — 연구 연계 1위
2. 다른 그림 찾기 — 연구 연계 2위
3. 순서 기억하기 — 연구 연계 3위 (현재 구현 완료 상태)
4. 신호에 맞춰 누르기 — 연구 연계 4위
5. 단어·그림 분류하기 — 연구 연계 5위
6. 일상생활 순서 맞추기 — 연구 연계 6위, 생활 맥락 적합성 높음

각 게임은 선정 이유, 관찰 가능한 인지 활동 영역, 게임별 데이터 구조 예시, 자녀 대시보드 연결 방식을 정의한다. 게임 결과는 의료적 진단이 아닌 개인별 활동 변화와 참여 패턴을 관찰하는 지표로 활용한다.

## 1. 기획 목적

시니어가 부담 없이 반복 참여할 수 있는 인지 활동 게임을 제공하고, 게임 수행 데이터를 부모님의 활동 변화와 참여 패턴을 이해할 수 있는 가족 케어 데이터로 연결한다.

게임은 의료적 진단이나 치매 여부를 판정하는 도구가 아니다. 정답률, 반응 시간, 오류 횟수, 재시도 여부, 참여 빈도 등을 개인별 변화 관찰 지표로 활용한다.

## 2. 게임 설계 기준

- 그림, 숫자, 단어, 생활 장면 등 익숙한 소재를 사용한다.
- 설명이 짧고 조작 방식이 직관적이어야 한다.
- 반복 수행과 난이도 조절이 가능해야 한다.
- 모든 게임 결과를 공통 데이터 구조로 저장할 수 있어야 한다.
- 한 번의 점수보다 개인별 반복 수행의 변화 흐름을 중요하게 본다.
- 게임 결과를 질병 진단이나 위험 판정으로 표현하지 않는다.

## 3. 게임 후보

### 3.1 순서 기억하기 (현재 구현 완료)

예시 블록을 순서대로 보여준 뒤 같은 순서로 선택하는 게임이다.

- 관찰 데이터: 정답률, 반응 시간, 오류 위치, 재시도 횟수, 힌트 사용 여부
- 기획 적합성: 기억력과 집중력 관련 수행 데이터를 만들기 쉽고 난이도 조절이 간단하다.

### 3.2 같은 그림 찾기

같은 그림 찾기는 카드 뒤집기 방식의 짝맞추기 게임이에요.

**게임 흐름**

1. 화면에 뒤집힌 카드 여러 장이 보인다.
2. 사용자가 한 번에 카드 두 장을 선택한다.
3. 두 카드 그림이 같으면 열린 상태로 유지되고, 다르면 잠시 뒤 다시 뒤집힌다.
4. 모든 짝을 찾으면 게임이 완료된다.

- 관찰 데이터: 오답 횟수, 카드 선택 시간, 완료 시간, 힌트 사용 여부
- 기획 적합성: 게임성이 높고 규칙이 직관적이며 시각적 기억 활동의 반복 데이터를 만들 수 있다.

### 3.3 단어·그림 분류하기

단어나 그림을 음식, 교통, 가족, 생활용품 등 올바른 범주로 분류하는 게임이다.

- 관찰 데이터: 범주별 정답률, 응답 시간, 오답 유형, 건너뛴 항목
- 기획 적합성: 언어 이해와 판단 활동을 포함하고, 선호 주제와 어려워하는 유형을 분석할 수 있다.

### 3.4 다른 그림 찾기

비슷한 그림 여러 개 중 서로 다른 부분을 찾는 게임이다.

- 관찰 데이터: 탐색 시간, 정답 여부, 오답 선택, 힌트 사용, 재시도
- 기획 적합성: 짧은 시간 안에 수행 결과가 명확하게 쌓이고 일일 미션으로 운영하기 쉽다.

### 3.5 일상생활 순서 맞추기

식사 준비, 외출 준비 등 친숙한 생활 장면을 올바른 순서로 배열하는 게임이다.

- 관찰 데이터: 순서 정답률, 누락 단계, 응답 시간, 힌트 사용 여부
- 기획 적합성: 시니어에게 친숙하고 게임 결과를 생활형 미션 추천과 가족 대시보드에 연결하기 좋다.

## 4. 추천 우선순위

### 4.1 구현 난이도·데이터 구조화 기준

```text
신호에 맞춰 누르기
→ 숫자·기호 짝맞추기
→ 다른 그림 찾기
→ 같은 그림 찾기
→ 순서 기억하기 (현재 구현 완료)
```

이 순위는 구현 복잡도, 짧은 플레이 시간, 정답·오답·반응 시간·재시도 데이터를 안정적으로 저장할 수 있는지를 기준으로 한다. `신호에 맞춰 누르기`는 가장 단순한 구조로 빠르게 검증할 수 있고, `순서 기억하기`는 데이터는 풍부하지만 상태 관리와 난이도 설계가 상대적으로 복잡하다.

### 4.2 인지훈련 연구와의 연계 기준

```text
숫자·기호 짝맞추기
→ 다른 그림 찾기
→ 순서 기억하기
→ 신호에 맞춰 누르기
→ 단어·그림 분류하기
→ 일상생활 순서 맞추기
```

이 순위는 게임 이름이 아니라 실제 플레이 구조가 연구 과제의 자극·반응·측정 방식과 얼마나 겹치는지를 기준으로 정한 기획 우선순위다. 숫자·기호 짝맞추기는 숫자-기호 대응과 처리 속도, 다른 그림 찾기는 시각 탐색과 식별, 순서 기억하기는 작업기억 과제와 구조적 유사성이 높다. 신호에 맞춰 누르기는 반응시간·선택적 주의·억제 측정과 연결되지만 단순한 한 버튼 버전은 연구용 처리속도 훈련과 동일하지 않다. 단어·그림 분류하기는 의미 기억·언어·범주화와, 일상생활 순서 맞추기는 순서화·계획·일상 문제 해결 활동과 연결되지만 두 게임 모두 특정 게임 자체의 훈련 효과 근거보다 인지 자극 활동으로서의 연계성이 상대적으로 약하다.

따라서 서비스에서는 게임을 의료적 진단이나 치매 예방 도구로 표현하지 않고, 인지 활동 참여와 개인별 수행 변화 관찰을 위한 미션으로 정의한다.

참고:

- [ACTIVE 인지훈련 무작위 대조시험](https://pubmed.ncbi.nlm.nih.gov/12425704/) — 시각 탐색·식별 기반 처리 속도, 기억, 추론 훈련
- [ACTIVE 처리 속도 훈련의 장기 추적 분석](https://pubmed.ncbi.nlm.nih.gov/24385640/) — 반복 훈련량과 처리 속도 수행 변화
- [노년층 작업기억 훈련 무작위 대조시험](https://pubmed.ncbi.nlm.nih.gov/29796016/) — 작업기억 훈련과 표적 탐지 수행
- [노년층 작업기억 훈련과 전략 사용 연구](https://pubmed.ncbi.nlm.nih.gov/28287787/) — 작업기억 과제와 처리 속도·기억 수행 변화
- [인지훈련이 치매 위험을 낮추는지에 대한 체계적 고찰](https://pubmed.ncbi.nlm.nih.gov/29255842/) — 인지훈련 효과를 과장하지 않기 위한 한계 근거

### 4.3 연구 연계 교차 검토 요약

| 게임 | 연구 과제와의 직접성 | 문서에서 사용할 수 있는 표현 | 설계 시 주의점 |
|---|---|---|---|
| 숫자·기호 짝맞추기 | 높음 | 숫자-기호 대응·처리 속도 과제에서 착안한 활동 | 키를 계속 보여주는지 숨기는지에 따라 처리 속도와 연합기억의 비중이 달라짐 |
| 다른 그림 찾기 | 중간~높음 | 시각 탐색·차이 식별 활동 | 시력, 색 대비, 그림 크기의 영향을 별도로 고려해야 함 |
| 순서 기억하기 | 높음 | 작업기억·순서 기억 활동 | 구현완료     |
| 신호에 맞춰 누르기 | 중간 | 반응시간·주의집중·반응 억제 활동 | 한 버튼 단순 반응인지, 선택 반응·누르지 않기 조건인지 구분해야 함 |
| 단어·그림 분류하기 | 중간 | 의미 연결·범주화 활동 | 단어 친숙도, 교육 수준, 언어·문화권 차이가 결과에 영향을 줄 수 있음 |
| 일상생활 순서 맞추기 | 중간 | 일상 장면의 순서화·계획 활동 | 실제 일상생활 수행 능력의 평가로 해석하지 않음 |

이 표의 직접성은 연구 과제와 게임의 구조가 닮았다는 의미이지, 해당 게임이 인지능력 향상이나 치매 예방 효과를 입증했다는 의미가 아니다.

## 5. 최종 MVP 후보 6종 상세

최종 후보는 인지훈련 연구에서 사용된 과제와의 구조적 유사성을 참고해 정리한다. 순위는 질환 진단·치매 예방 효과의 순위가 아니라, 게임의 자극·반응·측정 구조가 관련 인지 과제와 얼마나 닮았는지를 뜻한다.

### 5.1 숫자·기호 짝맞추기

**플레이 방식**  
화면 위 숫자-기호 규칙표를 보고, 아래에 제시되는 숫자에 맞는 기호를 빠르게 선택한다. 규칙표는 세션마다 바꿔 고정된 답을 외우는 효과를 줄인다.

**게임 흐름**

1. 이번 판의 숫자-기호 규칙표를 확인한다.
2. 숫자 하나가 제시되면, 규칙표에서 대응하는 기호를 찾아 선택한다.
3. 정답이면 다음 문제로 바로 넘어가고, 오답이면 정답을 짧게 보여준 뒤 다음 문제로 넘어간다.
4. 정해진 문제를 모두 풀면 정답률과 평균 응답 시간을 보여준다.

**재미·반복 요소**  
연속 정답, 개인 최고 기록, 새로운 기호 테마를 제공한다. 같은 규칙을 반복 암기하는 것이 아니라 매 세션 다른 규칙표를 사용한다.

**연관 인지 활동 영역**  
처리 속도, 시각적 주의력, 시각 탐색, 시각-운동 대응, 규칙 학습.

**저장 데이터 필드 [우리 데이터 구조 참고]**

```text
sessionId
problemCount
mappingVersion
correctCount
errorCount
meanResponseTimeMs
hintUsed
retryCount
accuracy
difficulty
completedAt
```

**자녀 대시보드 문구 예시**  
“이번 주 숫자·기호 활동에 3회 참여했어요.”  
“최근 정답률은 비슷하고, 문제를 푸는 시간은 조금 줄었어요.”

**난이도 조절**  
숫자·기호 종류 수, 규칙표 노출 방식, 문제 수, 제한 시간, 유사한 기호의 비율, 힌트 횟수를 조절한다.

**피해야 할 표현**  
“처리 속도가 저하됐습니다.”  
“기억력이 나빠졌습니다.”  
“인지 저하가 의심됩니다.”

**근거 수준 혹은 논문 출처**  
높음. 숫자-기호 대응 과제와 구조적으로 유사하지만, 앱 점수는 임상검사 점수가 아니다. [Digit Symbol 수행 연구](https://pubmed.ncbi.nlm.nih.gov/12675384/)

### 5.2 다른 그림 찾기

**플레이 방식**  
두 그림을 비교해 서로 다른 부분을 터치로 찾는다. 한 문제의 차이를 모두 찾으면 다음 장면으로 넘어간다.

**게임 흐름**

1. 좌우의 비슷한 생활 장면 그림을 함께 본다.
2. 서로 다른 부분을 찾으면 해당 위치를 터치한다.
3. 맞게 찾은 차이는 표시로 남고, 잘못 누르면 짧은 안내 후 다시 찾는다.
4. 모든 차이를 찾으면 다음 그림으로 넘어가며, 끝나면 완료 시간과 찾은 개수를 요약한다.

**재미·반복 요소**  
식탁, 산책, 반려동물 등 친숙한 테마를 순환하고, 연속 성공 시 작은 배지나 컬렉션을 제공한다.

**연관 인지 활동 영역**  
시각 탐색, 선택적 주의력, 시각적 변별, 처리 속도.

**저장 데이터 필드 [우리 데이터 구조 참고]**

```text
sessionId
problemCount
visualSearchSetSize
targetFoundCount
timeToTargetMs
falseSelectionCount
hintUsed
retryCount
accuracy
difficulty
completedAt
```

**자녀 대시보드 문구 예시**  
“이번 주 시각 활동에 3회 참여했어요.”  
“최근 정답률은 비슷하지만 문제를 해결하는 시간이 조금 줄었어요.”

**난이도 조절**  
그림 크기, 차이 개수, 배경 복잡도, 차이의 크기, 문제 수, 힌트 횟수를 조절한다.

**피해야 할 표현**  
“시력이 나빠졌습니다.”  
“공간지각 능력이 떨어졌습니다.”  
“인지 저하가 의심됩니다.”

**근거 수준 혹은 논문 출처**  
중간~높음. 시각 탐색·식별 기반 처리 속도 과제와 연결되지만, 일반적인 다른 그림 찾기 자체의 예방 효과를 뜻하지 않는다. [ACTIVE 무작위 대조시험](https://pubmed.ncbi.nlm.nih.gov/12425704/)

### 5.3 순서 기억하기 (현재 구현 완료)

**구현 상태**  
현재 MVP에서 구현이 완료된 게임이다. 이후에는 현 구현의 규칙과 저장 필드를 기준으로 난이도 조절·대시보드 문구를 연결한다.

**플레이 방식**  
블록이 순서대로 표시된 뒤, 사용자가 같은 순서로 다시 선택한다. MVP에 현재 구현 완료

**게임 흐름**

1. 여러 개의 블록이 차례로 강조 표시된다.
2. 표시가 끝나면 사용자가 기억한 순서대로 같은 블록을 선택한다.
3. 순서가 모두 맞으면 다음 단계로 넘어간다.
4. 순서가 틀리면 틀린 지점과 정답 순서를 짧게 확인한 뒤 같은 단계에 다시 도전하거나 다음 문제로 넘어간다.

**재미·반복 요소**  
성공하면 한 단계씩 순서 길이가 늘어나며, 테마별 그림 세트를 바꿔 새로움을 만든다.

**연관 인지 활동 영역**  
작업기억, 단기기억, 순서 기억, 집중력.

**저장 데이터 필드 [우리 데이터 구조 참고]**

```text
sessionId
problemCount
sequenceLength
correctSequenceCount
firstErrorIndex
meanResponseTimeMs
hintUsed
retryCount
accuracy
difficulty
completedAt
```

**자녀 대시보드 문구 예시**  
“이번 주 순서 기억 활동을 꾸준히 완료했어요.”  
“최근에는 이전보다 긴 순서의 문제에도 참여하고 있어요.”

**난이도 조절**  
순서 길이, 자극 노출 시간, 선택지 수, 그림 유사도, 방해 자극, 힌트 횟수를 조절한다.

**피해야 할 표현**  
“기억력이 떨어졌습니다.”  
“단기기억 저하가 확인됐습니다.”  
“치매 위험이 높습니다.”

**근거 수준 혹은 논문 출처**  
높음. 작업기억·순서 회상 과제와 직접 연결되지만, 장기적 전이 효과는 별도 검증이 필요하다. [노년층 작업기억 훈련 무작위 대조시험](https://pubmed.ncbi.nlm.nih.gov/29796016/)

### 5.4 신호에 맞춰 누르기

**플레이 방식**  
목표 신호가 나타날 때만 버튼을 누르고, 다른 신호에는 누르지 않는다. 단순 반응형보다 `누르기/누르지 않기` 조건을 포함해 게임 규칙을 명확히 한다. 목표 신호는 똑똑용돈의 아이콘을 넣는다. 똑똑 용돈의 아이콘이 나타나면 누르고 다른 그림이면 누르지 않기를 한다.

**게임 흐름**

1. 게임 시작 전 “똑똑용돈 그림이 나오면 누르고, 다른 그림이면 누르지 않기”처럼 목표 규칙을 알려준다.
2. 여러 신호가 화면에 하나씩 나타난다.
3. 목표 신호에만 반응하면 정답, 비목표 신호를 누르면 오반응, 목표 신호를 놓치면 누락으로 기록한다.
4. 모든 신호가 끝나면 정답 수·오반응·평균 반응 시간을 요약한다.

**재미·반복 요소**  
신호가 등장하는 리듬, 계절·날씨 테마, 연속 성공 기록을 활용한다. 너무 빠른 속도보다 이해 가능한 규칙과 성공 경험을 우선한다.

**연관 인지 활동 영역**  
반응 속도, 선택적 주의력, 지속적 주의력, 반응 억제.

**저장 데이터 필드 [우리 데이터 구조 참고]**

```text
sessionId
targetCount
correctCount
falsePositiveCount
omissionCount
meanReactionTimeMs
retryCount
accuracy
difficulty
completedAt
```

**자녀 대시보드 문구 예시**  
“오늘 집중 신호 활동을 완료했어요.”  
“최근 참여 횟수와 완료율이 지난주와 비슷해요.”

**난이도 조절**  
신호 등장 간격, 목표·비목표 신호 비율, 신호 종류 수, 화면 위치, 문제 수, 힌트 횟수를 조절한다.

**피해야 할 표현**  
“집중력이 저하됐습니다.”  
“반응 능력이 떨어졌습니다.”  
“실행기능 문제가 의심됩니다.”

**근거 수준 혹은 논문 출처**  
중간. 반응시간·주의·억제 과제와 개념적으로 연결되지만, 단순 신호 게임의 훈련 효과를 주장하지 않는다. [노년층 억제·작업기억 훈련 무작위 대조시험](https://pubmed.ncbi.nlm.nih.gov/33343503/)

### 5.5 단어·그림 분류하기

**플레이 방식**  
단어나 그림을 음식, 교통, 가족, 생활용품처럼 알맞은 범주에 넣는다. 읽기 부담을 줄이기 위해 그림 중심으로 시작하고 단어는 보조로 사용한다.

**게임 흐름**

1. 화면에 두세 개의 범주 상자가 보인다. 예: `음식`, `교통`.
2. 그림 또는 단어 하나가 제시되면 알맞은 범주 상자로 끌어 놓거나 터치해 선택한다.
3. 정답이면 해당 범주에 쌓이고, 오답이면 쉬운 설명과 함께 다시 고를 수 있다.
4. 모든 항목을 분류하면 범주별 정답률과 참여 결과를 보여준다.

**재미·반복 요소**  
사용자가 좋아하는 음식·여행·추억 테마를 순환하고, 범주 완성 컬렉션을 제공한다.

**연관 인지 활동 영역**  
의미 기억, 언어 이해, 범주화, 추론과 판단.

**저장 데이터 필드 [우리 데이터 구조 참고]**

```text
sessionId
problemCount
categoryCount
correctCount
errorCategory
meanResponseTimeMs
skippedCount
hintUsed
accuracy
difficulty
completedAt
```

**자녀 대시보드 문구 예시**  
“이번 주 단어·그림 활동에 2회 참여했어요.”  
“익숙한 생활 주제의 활동을 편안하게 완료했어요.”

**난이도 조절**  
범주 수, 항목 수, 그림과 단어의 동시 노출 여부, 범주 간 유사도, 제한 시간, 힌트 횟수를 조절한다.

**피해야 할 표현**  
“언어 능력이 떨어졌습니다.”  
“의미기억 저하가 보입니다.”  
“치매 관련 언어 문제가 의심됩니다.”

**근거 수준 혹은 논문 출처**  
중간. 의미 처리·범주화 활동과 연결되지만, 친숙도·교육 수준·언어권의 영향이 크다. [노년층 단어-그림 의미 처리 연구](https://pubmed.ncbi.nlm.nih.gov/29180957/)

### 5.6 일상생활 순서 맞추기

**플레이 방식**  
식사 준비, 외출 준비, 약 챙기기처럼 친숙한 생활 장면의 카드를 올바른 순서로 배열한다. 정답을 맞힌 뒤에는 짧은 생활 팁이나 완성된 장면을 보여준다.

**게임 흐름**

1. “외출 준비하기”처럼 오늘의 생활 장면과 섞여 있는 단계 카드가 제시된다.
2. 사용자가 카드를 순서대로 탭하거나 끌어 놓아 이야기를 완성한다.
3. 맞게 놓인 단계는 유지하고, 순서가 맞지 않으면 힌트로 앞·뒤 관계를 알려준다.
4. 모든 단계가 완성되면 완성된 생활 장면과 참여 결과를 보여준다.

**재미·반복 요소**  
아침·외출·장보기·가족 방문처럼 일상 테마를 바꾸고, 완성한 생활 장면을 앨범처럼 모은다. 가족이 자주 언급한 생활 주제를 다음 미션에 반영할 수 있다.

**연관 인지 활동 영역**  
순서화, 계획, 판단, 작업기억, 일상 문제 해결.

**저장 데이터 필드 [우리 데이터 구조 참고]**

```text
sessionId
scenarioId
stepCount
correctOrderCount
firstErrorStep
meanResponseTimeMs
skippedStepCount
hintUsed
retryCount
accuracy
difficulty
completedAt
```

**자녀 대시보드 문구 예시**  
“이번 주 생활 순서 활동에 3회 참여했어요.”  
“최근에는 더 많은 단계의 생활 장면을 끝까지 완성했어요.”

**난이도 조절**  
단계 수, 장면의 친숙도, 선택지 수, 유사한 단계의 포함 여부, 설명 문구 노출, 힌트 횟수를 조절한다.

**피해야 할 표현**  
“일상생활 능력이 떨어졌습니다.”  
“약 복용 관리가 어렵습니다.”  
“독립생활이 어렵다고 판단됩니다.”

**근거 수준 혹은 논문 출처**  
중간. 순서화·계획 활동과 연결되며, 일상생활 수행에는 기억·판단·조직화가 함께 필요하다. 다만 게임 결과를 실제 생활 수행 능력의 평가로 사용하지 않는다. [노년층 IADL과 인지 기능 관계 연구](https://pmc.ncbi.nlm.nih.gov/articles/PMC12033700/)

### 5.7 데이터 해석 기준

각 게임의 결과는 의료적 평가나 치매 위험 판정에 사용하지 않는다. 개인의 과거 수행 기록을 기준으로 참여 빈도, 완료율, 반응 시간, 정답률, 오류와 재시도 변화를 비교하고, 이를 자녀 대시보드의 활동 요약과 다음 미션 추천에 활용한다.

```text
게임별 수행 데이터
→ 개인별 기준선과 최근 기록 비교
→ 참여·수행 변화 요약
→ 맞춤 미션 추천
```

## 6. 공통 데이터 구조

게임마다 다른 수행 결과를 다음 공통 필드로 구조화한다.

```text
activityId
activityType
difficulty
sessionId
startedAt
completedAt
completionStatus
accuracy
responseTimeMs
errorCount
retryCount
hintUsed
skippedCount
score
responseSummary
```

MVP에서는 게임 중 발생하는 모든 이벤트를 저장하기보다, 게임 완료 시 핵심 수행 결과를 요약 저장한다.

## 7. 가족 대시보드 연결 데이터

원시 게임 로그가 아니라 다음과 같은 요약 정보를 자녀에게 제공한다.

- 오늘의 미션 참여 여부
- 이번 주 참여 횟수와 완료율
- 평균 수행 시간 변화
- 정답률과 오류·재시도 변화
- 선호하는 게임 유형
- 최근 활동 증가·감소 흐름
- 다음 추천 미션

## 8. 플라이휠 구조

```text
게임 참여
→ 수행 데이터 저장
→ 개인별 변화 비교
→ AI 활동 요약
→ 자녀 가족 리포트
→ 맞춤 미션 추천
→ 재참여
```

## 9. 현재 가설

- 최종 후보는 6개 게임으로 구성한다.
- MVP에서는 후보 중 2~3개를 우선 구현하고 나머지는 확장 후보로 둔다.
- 게임 선택 방식과 당일 미션 자동 부여 방식은 기획 변경 가능성을 열어둔다.
- 데이터는 의료적 평가가 아니라 참여와 수행 변화 관찰에 사용한다.
