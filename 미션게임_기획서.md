# 똑똑용돈 게임 기획 기록

## 최종 MVP 후보 5종 요약

최종 MVP 후보는 구현 난이도 기준 상위 2개와 인지훈련 연구와의 연계 기준 상위 3개를 결합해 선정한다.

1. 신호에 맞춰 누르기 — 구현 난이도 1위
2. 숫자·기호 짝맞추기 — 구현 난이도 2위
3. 다른 그림 찾기 — 연구 연계 1위
4. 순서 기억하기 — 연구 연계 2위
5. 단어·그림 분류하기 — 연구 연계 3위

각 게임은 선정 이유, 관찰 가능한 인지 활동 영역, 게임별 데이터 구조 예시, 자녀 대시보드 연결 방식을 정의한다. 게임 결과는 의료적 진단이 아닌 개인별 활동 변화와 참여 패턴을 관찰하는 지표로 활용한다.

## 1. 기획 목적

시니어가 부담 없이 반복 참여할 수 있는 인지 활동 게임을 제공하고, 게임 수행 데이터를 부모님의 활동 변화와 참여 패턴을 이해할 수 있는 가족 케어 데이터로 연결한다.

게임은 의료적 진단이나 치매 여부를 판정하는 도구가 아니다. 정답률, 반응 시간, 오류 횟수, 재시도 여부, 참여 빈도 등을 개인별 변화 관찰 지표로 활용한다.

## 2. 게임 설계 기준

- 한 번에 1~3분 안에 완료할 수 있어야 한다.
- 그림, 숫자, 단어, 생활 장면 등 익숙한 소재를 사용한다.
- 설명이 짧고 조작 방식이 직관적이어야 한다.
- 반복 수행과 난이도 조절이 가능해야 한다.
- 모든 게임 결과를 공통 데이터 구조로 저장할 수 있어야 한다.
- 한 번의 점수보다 개인별 반복 수행의 변화 흐름을 중요하게 본다.
- 게임 결과를 질병 진단이나 위험 판정으로 표현하지 않는다.

## 3. 게임 후보

### 3.1 순서 기억하기

그림, 숫자 또는 생활용품을 순서대로 보여준 뒤 같은 순서로 선택하는 게임이다.

- 관찰 데이터: 정답률, 반응 시간, 오류 위치, 재시도 횟수, 힌트 사용 여부
- 기획 적합성: 기억력과 집중력 관련 수행 데이터를 만들기 쉽고 난이도 조절이 간단하다.

### 3.2 같은 그림 찾기

여러 장의 카드 중 같은 그림의 짝을 찾는 게임이다.

- 관찰 데이터: 오답 횟수, 카드 선택 시간, 완료 시간, 힌트 사용 여부
- 기획 적합성: 게임성이 높고 규칙이 직관적이며 시각적 기억 활동의 반복 데이터를 만들 수 있다.

### 3.3 단어·그림 분류하기

단어나 그림을 음식, 교통, 가족, 생활용품 등 올바른 범주로 분류하는 게임이다.

- 관찰 데이터: 범주별 정답률, 응답 시간, 오답 유형, 건너뛴 항목
- 기획 적합성: 언어 이해와 판단 활동을 포함하고, 선호 주제와 어려워하는 유형을 분석할 수 있다.

### 3.4 다른 그림 찾기

비슷한 그림 여러 개 중 서로 다른 부분을 찾는 게임이다.

- 관찰 데이터: 탐색 시간, 정답 여부, 오답 선택, 힌트 사용, 재시도
- 기획 적합성: 짧은 시간 안에 수행 결과가 명확하게 쌓이고 일일 미션으로 운영하기 쉽다.

### 3.5 일상생활 순서 맞추기

식사 준비, 외출 준비 등 친숙한 생활 장면을 올바른 순서로 배열하는 게임이다.

- 관찰 데이터: 순서 정답률, 누락 단계, 응답 시간, 힌트 사용 여부
- 기획 적합성: 시니어에게 친숙하고 게임 결과를 생활형 미션 추천과 가족 대시보드에 연결하기 좋다.

## 4. 추천 우선순위

### 4.1 구현 난이도·데이터 구조화 기준

```text
신호에 맞춰 누르기
→ 숫자·기호 짝맞추기
→ 다른 그림 찾기
→ 같은 그림 찾기
→ 순서 기억하기
```

이 순위는 구현 복잡도, 짧은 플레이 시간, 정답·오답·반응 시간·재시도 데이터를 안정적으로 저장할 수 있는지를 기준으로 한다. `신호에 맞춰 누르기`는 가장 단순한 구조로 빠르게 검증할 수 있고, `순서 기억하기`는 데이터는 풍부하지만 상태 관리와 난이도 설계가 상대적으로 복잡하다.

### 4.2 인지훈련 연구와의 연계 기준

```text
신호에 맞춰 누르기
→ 다른 그림 찾기
→ 순서 기억하기
→ 단어·그림 분류하기
→ 오늘 할 일 기억하기
```

이 순위는 주의력, 처리 속도, 기억력, 추론 활동과의 연계성 및 관련 연구를 참고한 기획 우선순위다. 특히 처리 속도 훈련은 ACTIVE 연구에서 장기 추적 근거가 확인된 영역과 연결되지만, 단순한 버튼 게임 자체가 치매 예방 효과를 가진다는 의미는 아니다.

따라서 서비스에서는 게임을 의료적 진단이나 치매 예방 도구로 표현하지 않고, 인지 활동 참여와 개인별 수행 변화 관찰을 위한 미션으로 정의한다.

참고:

- [ACTIVE Cognitive Training and Rates of Incident Dementia](https://pmc.ncbi.nlm.nih.gov/articles/PMC3384749/)
- [NIH: Cognitive speed training and long-term dementia diagnosis follow-up](https://www.nih.gov/news-events/news-releases/cognitive-speed-training-over-weeks-may-delay-diagnosis-dementia-over-decades)
- [Does Cognitive Training Prevent Cognitive Decline?](https://pubmed.ncbi.nlm.nih.gov/29255842/)

## 5. 최종 MVP 후보 5종

최종 후보는 구현 난이도 기준 상위 2개와 인지훈련 연구와의 연계 기준 상위 3개를 결합해 선정한다. 순위가 높다는 것은 해당 게임이 특정 질환을 진단하거나 치매 예방 효과가 확정되었다는 의미가 아니라, MVP의 구현·데이터화·인지 활동 설계에 적합하다는 의미다.

### 5.1 신호에 맞춰 누르기

- 선정 기준: 구현 난이도 1위
- 추천 이유: 단일 화면과 버튼 입력만으로 구현할 수 있어 가장 빠르게 검증할 수 있다. 정답·오답·반응 시간·누락 데이터를 안정적으로 만들 수 있다.
- 인지 활동 영역: 처리 속도, 선택적 주의력, 지속적 주의, 반응 억제
- 데이터 구조 예시:

```text
targetCount
correctCount
falsePositiveCount
omissionCount
meanReactionTimeMs
difficulty
retryCount
```

### 5.2 숫자·기호 짝맞추기

- 선정 기준: 구현 난이도 2위
- 추천 이유: 숫자 또는 기호를 짝짓는 단순한 규칙으로 구성할 수 있고, 문제 수와 제한 시간으로 난이도를 조절하기 쉽다.
- 인지 활동 영역: 시각적 주의력, 처리 속도, 작업 기억, 규칙 학습
- 데이터 구조 예시:

```text
itemCount
correctCount
errorCount
responseTimeMs
hintUsed
completionStatus
```

### 5.3 다른 그림 찾기

- 선정 기준: 인지훈련 연구와의 연계 1위
- 추천 이유: 시각적 정보를 빠르게 탐색하고 차이를 찾는 구조라 처리 속도·주의력 훈련과 연결하기 좋다. 이미지 세트만 교체해 반복 미션으로 운영할 수 있다.
- 인지 활동 영역: 시각 탐색, 선택적 주의, 처리 속도, 시각적 변별
- 데이터 구조 예시:

```text
visualSearchSetSize
targetFound
timeToTargetMs
falseSelections
hintUsed
retryCount
```

### 5.4 순서 기억하기

- 선정 기준: 인지훈련 연구와의 연계 2위
- 추천 이유: 기억력 훈련과 직접적으로 연결되며, 항목 수와 노출 시간으로 개인별 난이도를 조정할 수 있다. 반복 수행을 통해 기억 수행의 변화 흐름을 관찰하기 좋다.
- 인지 활동 영역: 작업 기억, 단기 기억, 순서 기억, 집중력
- 데이터 구조 예시:

```text
sequenceLength
correctSequenceCount
firstErrorIndex
responseTimeMs
hintUsed
retryCount
```

### 5.5 단어·그림 분류하기

- 선정 기준: 인지훈련 연구와의 연계 3위
- 추천 이유: 단어나 그림을 범주화하는 방식으로 구성해 언어·추론 활동을 함께 다룰 수 있다. 음식, 가족, 생활용품 등 친숙한 주제를 사용하면 시니어의 참여 부담을 낮출 수 있다.
- 인지 활동 영역: 의미 기억, 언어 이해, 범주화, 추론과 판단
- 데이터 구조 예시:

```text
categoryCount
accuracyByCategory
responseTimeMs
errorCategory
skippedCount
completionStatus
```

### 5.6 데이터 해석 기준

각 게임의 결과는 의료적 평가나 치매 위험 판정에 사용하지 않는다. 개인의 과거 수행 기록을 기준으로 참여 빈도, 완료율, 반응 시간, 정답률, 오류와 재시도 변화를 비교하고, 이를 자녀 대시보드의 활동 요약과 다음 미션 추천에 활용한다.

```text
게임별 수행 데이터
→ 개인별 기준선과 최근 기록 비교
→ 참여·수행 변화 요약
→ 맞춤 미션 추천
```

## 6. 공통 데이터 구조

게임마다 다른 수행 결과를 다음 공통 필드로 구조화한다.

```text
activityId
activityType
difficulty
sessionId
startedAt
completedAt
completionStatus
accuracy
responseTimeMs
errorCount
retryCount
hintUsed
skippedCount
score
responseSummary
```

MVP에서는 게임 중 발생하는 모든 이벤트를 저장하기보다, 게임 완료 시 핵심 수행 결과를 요약 저장한다.

## 7. 가족 대시보드 연결 데이터

원시 게임 로그가 아니라 다음과 같은 요약 정보를 자녀에게 제공한다.

- 오늘의 미션 참여 여부
- 이번 주 참여 횟수와 완료율
- 평균 수행 시간 변화
- 정답률과 오류·재시도 변화
- 선호하는 게임 유형
- 최근 활동 증가·감소 흐름
- 다음 추천 미션

## 8. 플라이휠 구조

```text
게임 참여
→ 수행 데이터 저장
→ 개인별 변화 비교
→ AI 활동 요약
→ 자녀 가족 리포트
→ 맞춤 미션 추천
→ 재참여
```

## 9. 현재 가설

- 최종 후보는 5개 게임으로 구성한다.
- MVP에서는 후보 중 2~3개를 우선 구현하고 나머지는 확장 후보로 둔다.
- 게임 선택 방식과 당일 미션 자동 부여 방식은 기획 변경 가능성을 열어둔다.
- 데이터는 의료적 평가가 아니라 참여와 수행 변화 관찰에 사용한다.
