### 기술 스택 

|영역|내용|
|---|---|
|iOS|**Swift + SwiftUI**|
|음성 녹음·재생|**AVFoundation** (음성 일기·응원 편지 녹음 AAC ≤60초, 편지 재생)|
|음성 STT|**iOS Speech framework(ko-KR) — 기기 내(on-device) 전사 강제**. 음성 일기(시니어)·응원 편지(자녀) 공용. 음성 일기 오디오는 서버 미전송(API_SPEC §1.9 특칙), on-device 미지원 기기 처리 방침은 A-7 ⑦⒜ 결정 대기|
|결제|**StoreKit2** (포인트 충전·프리미엄 구독 — 서버 트랜잭션 검증, 카드 정보 미수집)|
|Backend|**Python + FastAPI + PostgreSQL(pgvector)**|
|배포 (운영 적용)|**AWS EC2(Docker Compose) + RDS PostgreSQL 16** — 저장 암호화(RDS·EBS), 도메인 TLS 종단(공인 인증서 자동 갱신), 시크릿은 SSM SecureString. 접속 정보는 git 밖 로컬 파일 참조|
|AI (운영 적용)|**Gemini 3.5 Flash(구조화) + gemini-embedding-001(임베딩 768차원) + Qwen 3 32B(추천·리포트, OpenAI 호환 API)** — OpenAI는 용도 미정(A-7 ⑤)|
|CI/CD|**GitHub Actions**|
|개발 도구|**Claude Code, Codex**|
|UI/UX|**Claude Design, Figma**|
|데이터|활동 데이터 저장 구조 + 개인정보·활동 데이터 분리 등 기본 보안 원칙 반영|


### AI 모델 조합

|태스크|모델/방식|왜 이 선택?|
|---|---|---|
|활동 데이터 구조화 (하루 여러 번)|**Gemini 3.5 Flash** (Google AI Studio 무료 티어)|제일 자주 도는 작업이라 무료 티어 요청 한도가 관대한 Flash 계열로. 2.5 Flash는 신규 발급 키에서 생성 불가("no longer available to new users") 실측 — 3.5로 확정. 모델명은 env로 교체 가능|
|추천 후보 임베딩|**gemini-embedding-001, 768차원 축소**(outputDimensionality)+정규화|text-embedding-004가 신규 키 모델 목록에서 제거됨 — pgvector 768차원 유지가 가능한 후속 모델로 확정|
|변화 패턴 분석|통계 로직 (코드)|숫자 계산은 AI보다 코드가 더 정확하고 공짜예요|
|맞춤 활동 추천 (하루 1회)|Qwen 3 32B (**OpenAI 호환 API — 운영은 OpenRouter**, 엔드포인트 env 교체로 Groq 전환 가능)|매일 도는 개인화 로직엔 충분한 품질. 보유 키가 OpenRouter라 운영 엔드포인트로 확정, 코드가 아닌 설정으로 호스트 전환|
|가족 리포트 (주 1회)|Qwen 3 32B (OpenAI 호환 API — 추천과 동일 엔드포인트)|가족이 직접 읽는 민감한 글이라 원래는 상위 모델을 썼지만, 무료 조합에서는 같은 Qwen을 쓰되 프롬프트로 비진단 톤을 강하게 강제하고, 데모 전 사람이 직접 출력을 검수하는 과정을 꼭 추가|

전 태스크 공통: 키 미설정·호출 실패 시 결정적 스텁 폴백으로 데모 무중단. 운영 검증: 구조화·임베딩·추천·리포트 4개 경로 모두 실모델 성공 이력을 `ai_jobs`로 확인.

### 음성 STT·음성 분석 도구 선택

|태스크|도구/방식|왜 이 선택?|
|---|---|---|
|음성 일기·응원 편지 전사 (STT)|**iOS Speech framework, ko-KR — on-device 강제**(`requiresOnDeviceRecognition`)|음성 원본을 기기 밖으로 내보내지 않는 보안 원칙(§1.9 특칙)을 아키텍처로 보장하는 유일한 선택지. 무료·오프라인 동작·별도 서버 비용 없음. 서버 STT(Whisper·클라우드 STT)는 오디오 업로드가 전제라 배제. on-device 미지원 기기의 처리(기능 비활성 vs Apple 서버 STT 허용)는 A-7 ⑦⒜ 결정 대기|
|기기 측 발화 요약 수치 (발화 시간·속도·멈춤·재시작·STT 신뢰도)|**iOS 클라이언트 코드** — STT 타임스탬프 기반 산출|원본 오디오 없이도 산출 가능한 수치는 기기에서 계산해 전송(문항 요약만 보내는 게임 계측과 동일 원칙)|
|음성 일기 발화 특징 분석 (반복 문장 비율·어휘 다양성·필러 빈도·조리성 신호 변화 추이)|**통계 로직(코드), 일 배치** — 외부 전송 없음|수치 산출은 AI보다 코드가 정확하고 무료. 민감도 최상 데이터라 외부 반출 최소화가 우선|
|음성 일기 정서·주제 구조화 (하루 이야기 요약·기분 신호·개연성 신호)|**Gemini 3.5 Flash — 조건부 비활성**: 입력을 학습에 쓰지 않는 유료 티어/약관 확인(A-7 ⑥) 전까지 미운영, 활성화 시에도 PII 스크러빙+가명 ID 필수|자유 발화 전사는 무료 티어(입력 학습 가능 약관)로 보낼 수 없음 — 약관 확인 전까지는 위의 통계 지표만 운영|
|음성 응원 편지|**프로파일링 분석 미적용**|전달 콘텐츠 — 발화·정서 분석 투입 금지(보안 문서.md §4.3 L-3)|
|응원 편지 안전 필터 (욕설·모욕·위협·위험 표현·금전 요구 의심 판별)|**서버 내 코드 — 금칙어·패턴 사전** (+ iOS 로컬 사전 1차 검사)|편지 내용을 외부 AI에 보내는 것 자체가 프라이버시 위반이라 외부 모델 배제. 감지 시 저장 없이 발송 거부, 로그는 통과/차단만(보안 문서.md §4.3 L-7)|
